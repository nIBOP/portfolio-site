# Структура стилей проекта

## Организация файлов

### Глобальные стили
- **`App.css`** - Базовые стили, общие компоненты (карточки, кнопки, модальные окна), layout
- **`index.css`** - Глобальные стили и переменные

### Стили страниц
- **`App.home.css`** - Стили для главной страницы (hero, специализации, timeline, резюме)
- **`pages/Blog.css`** - Стили для страницы списка статей
- **`pages/BlogPost.css`** - Стили для страницы отдельной статьи

### Стили компонентов
- **`components/Timeline.css`** - Стили для компонента Timeline
- **`components/PdfViewer.tsx`** - PDF вьювер (без отдельного CSS файла)

## Принципы организации

### 1. Минимум inline стилей в TSX
- ✅ Используются только для динамических значений (цвета тегов)
- ❌ Все остальные стили вынесены в CSS файлы

### 2. Модульность
- Каждая страница имеет свой CSS файл
- Каждый компонент имеет свой CSS файл
- Общие стили находятся в App.css

### 3. Префиксы классов
- `.home-page` - главная страница
- `.blog-page` - список статей
- `.blog-post-page` - отдельная статья
- `.timeline-container` - компонент timeline

### 4. Адаптивность
Все медиа-запросы организованы по брейкпоинтам:
- `> 1170px` - десктоп (полная версия timeline)
- `720px - 1170px` - планшет
- `< 720px` - мобильные устройства (центрирование контента)
- `< 520px` - малые экраны
- `< 450px` - очень малые экраны

### 5. Выравнивание контента

#### Главная страница
- **> 720px**: все слева
- **< 720px**: заголовки и кнопки по центру

#### Блог (список)
- **> 720px**: все слева
- **< 720px**: заголовки по центру, контент карточек слева

#### Статья блога
- **> 720px**: все слева
- **< 720px**: заголовок по центру, текст статьи слева

## Используемые классы

### Общие компоненты (App.css)
- `.page` - основной контейнер страницы
- `.section` - секция контента
- `.card` / `.card--wide` - карточки
- `.primary-button` / `.secondary-button` - кнопки
- `.modal` - модальные окна
- `.hero` - блок с аватаром
- `.two-column-layout` - двухколоночная структура

### Главная страница (App.home.css)
- `.home-page` - контейнер главной
- `.specializations` - блок специализаций
- `.spec-buttons` - кнопки специализаций
- `.resume` - блок резюме

### Блог (Blog.css)
- `.blog-page` - контейнер страницы блога
- `.tag-filter-container` - фильтр по тегам
- `.tag-link` - ссылка тега
- `.card-meta` - метаинформация статьи
- `.card-tags` - теги в карточке
- `.read-button` - кнопка "Читать"

### Статья (BlogPost.css)
- `.blog-post-page` - контейнер страницы статьи
- `.post-title` - заголовок статьи
- `.post-meta` - метаинформация
- `.post-tags` - теги статьи
- `.post-content` - контент статьи
- `.back-button` - кнопка возврата

### Timeline (Timeline.css)
- `.timeline-container` - контейнер timeline
- `.timeline` - сам timeline
- `.event` - событие в timeline

## Правила работы со стилями

1. **Не добавляйте inline стили** без крайней необходимости
2. **Используйте существующие классы** перед созданием новых
3. **Группируйте стили логически** (базовые → адаптивные)
4. **Комментируйте** сложные секции
5. **Тестируйте** на всех брейкпоинтах
